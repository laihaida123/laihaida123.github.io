1. 明确创建智能体的基本流程

以下是在扣子（coze.com）平台创建智能体的全流程：需求分析→Prompt设计→工具与知识库配置（可能涉及RAG、Function Calling）→测试与迭代。

2. 需求分析

在创建智能体之前，你要明确智能体的目标。

比如你想做一个“美食推荐助手”，那么你需要思考：

  • 用户是谁：普通食客、美食爱好者等。
  • 需要解决什么问题：根据用户的口味偏好、预算、地点等信息，推荐合适的餐厅和美食。

3. Prompt设计

3.1 Prompt三要素讲解

  • 角色定位（“我是…”）：明确智能体扮演的角色，例如“我是一位专业的美食推荐助手”。
  • 任务描述（“帮我…”）：描述智能体需要完成的具体任务，如“帮用户根据其输入的口味偏好、预算和地点，推荐合适的餐厅和美食”。 - 规则约束（“需…”）：对输出进行一些限制，比如“需提供餐厅的名称、地址、联系电话以及推荐菜品”。

4. 工具与知识库配置

4.1 基础工具选用

扣子平台自带一些基础工具，如“网页搜索”“知识库问答”“文件解析”等。以美食推荐助手为例，如果你想获取最新的餐厅信息，可以启用“网页搜索”工具。

  • 启用方法：在平台的工具配置界面，找到“网页搜索”工具并勾选。
  • 参数设置：可以设置搜索关键词，例如根据用户输入的地点和美食类型生成搜索关键词，如“北京川菜餐厅”。

4.2 RAG（检索增强生成）的利用与原理

  • 原理：RAG就像是一个聪明的图书管理员。当你向智能体提问时，它不会立刻回答，而是先去一个大的“知识库图书馆”里查找相关的信息，然后把找到的信息和自己的知识结合起来，生成一个准确的回答。简单来说，就是先检索知识，再用这些知识来生成回答。
  • 在扣子平台的利用：你可以上传本地的知识库文件（如CSV文件，包含餐厅信息），或者使用平台的“知识库问答”工具。上传后，智能体在回答问题时会优先从这个知识库中查找相关信息。

4.2.2 核心四步走：从文档到知识库

整个过程分为四大步，每一步都很关键。下图为你清晰展示了这个流程的全貌：

flowchart TD
    subgraph A[第一步：原料准备]
        A1[原始文档] --> A2[格式解析] --> A3[清洗整理]
    end

    subgraph B[第二步：知识切片]
        B1[长文本] --> B2{选择切片方式}
        B2 --> B3[固定长度]
        B2 --> B4[按段落/标题]
        B3 & B4 --> B5[大小适中的文本块]
    end

    subgraph C[第三步：赋予“记忆点”]
        C1[文本块] --> C2[向量化模型<br/>将文字转为数学向量] --> C3[向量<br/>一串有含义的数字]
    end

    subgraph D[第四步：入库上架]
        C3 --> D1[向量数据库] --> D2[索引与存储]
    end

    A --> B --> C --> D

下面，我们来详细拆解每一步。

第一步：文档准备 —— 准备“原料”

目标：把各种格式的原始文件，变成干净、统一的纯文本。

  1. 文档类型：

    • 纯文本：.txt, .md 等，最简单。
    • 办公文档：.pdf, .docx, .pptx, .xlsx。需要工具解析文字和表格。
    • 网页：.html。需要去掉导航栏、广告等噪音，只留核心内容。
    • 多媒体：音频（转成文字稿）、图片（用OCR识别文字）。

  2. 关键操作：

    • 加载：用代码库（如 PyPDF2, python-docx, BeautifulSoup）读取文件内容。
    • 提取：从文件中把文字“抠”出来。
    • 清洗：删除乱码、无关的页眉页脚、广告语、特殊符号。
    • 标准化：统一换行符、空格等。

新手建议：先从简单的 .txt 或 .md 文件开始，避免复杂的格式解析问题。

第二步：文档切片 —— 把书拆成“词条”

目标：把一本厚厚的“书”（长文档），切成一段段大小合适的“词条”（文本块），方便大脑快速查找。

为什么必须切？

  • 大模型一次能处理的文字长度有限。
  • 检索时，直接匹配短文本块比匹配整本书更准、更快。

两种主要的切片方式：

  1. 固定长度切片（最常用）

    • 怎么做：像用尺子量一样，每 N 个字符或单词就切一刀。
    • 优点：简单，可控。
    • 缺点：可能从一句话中间切断，破坏语义。
    • 关键参数：
      • chunk_size：块大小。新手建议设为 500-1000 字。太小则信息不全，太大则检索不精准。
      • chunk_overlap：块重叠。建议设为大小的 10%-20%。比如切到第500字时，回头包含前50字。这能防止上下文的联系被切断，是非常重要的技巧！

  2. 智能切片（更优）

    • 怎么做：尽量在自然边界处切割，比如：
      • 按段落切：遇到两个换行符 \n\n 就切。
      • 按句子切：遇到句号、问号就切。
      • 按标题切：根据 # 标题 来划分章节。
    • 优点：保持语义完整性，效果通常更好。
    • 工具：可以直接使用 LangChain 库的 RecursiveCharacterTextSplitter，它会智能地按 [“\n\n”, “\n”, “。”, “.”, “ ”] 这样的优先级去尝试切割。
第三步：文档向量化 —— 给词条编上“智能条形码”

目标：让计算机能“理解”和“比较”这些文字的意思。

  1. 什么是向量化？
    把一段文字，通过一个叫做 “嵌入模型” 的AI，转换成一串有意义的数字（比如1536个数字）。这个过程就叫“向量化”，生成的那串数字就是“向量”或“嵌入”。

    • 关键：意思相似的文本，它们的向量在数学上（比如计算余弦相似度）也会很接近。

    举个例子：

    • “我喜欢吃苹果” → 向量 [0.1, 0.5, -0.3, …]
    • “我爱食用水果” → 向量 [0.12, 0.48, -0.28, …]
    • 这两个向量会非常接近，因为它们意思相似。

  2. 怎么做？
    调用一个嵌入模型的API或本地服务。

第四步：存储到向量数据库 —— 建立“智能索引”

目标：把所有切好并转成向量的文本块，用一种特殊的数据库存起来，以便进行闪电般的相似度搜索。

  1. 为什么用向量数据库？
    传统数据库按关键词搜索（比如搜索“苹果”）。向量数据库能按意思搜索（你问“有什么水果推荐？”，它能把“苹果”、“香蕉”的文本块都找出来）。

  2. 过程：

    • 将上一步生成的 （向量，对应的文本块，元数据） 一起存入向量数据库。
    • 元数据可以包括：这个块来自哪个文件、第几页、什么时间创建的等，方便后期筛选。

  3. 向量数据库 就是一个超级智能的图书管理员，你问他任何问题（也转换成智能标签），他都能立刻找到意思最相关的几页词条给你。

4.2.3 在coze上进行知识库搭建

在Coze（扣子）上构建知识库非常简单，你只需要准备好文档，平台会为你自动完成复杂的处理工作（如文档切片、向量化）。整个过程就像给你的AI助手上传一本参考书。

📝 详细操作指南

下面我们展开上图中的关键步骤，手把手教你操作。

1. 进入创建页面
登录Coze官网后，你需要找到创建入口。主要有两种方式：

  • 推荐路径：点击左侧栏的「工作空间」，进入你的「个人空间」，然后在「资源库」页面找到并点击右上角的 “+ 资源” 按钮，选择 “知识库” 即可开始创建。
  • 另一种方式是，你也可以在创建或编辑一个 “Bot”（智能体） 时，在「技能」配置区找到并点击“添加知识库”。

2. 填写信息与添加文档
创建时需要为知识库命名，并选择添加文档的方式。根据你的资料类型，选择最适合的方法：

添加方式 适用场景 操作简介
上传本地文件 拥有PDF、Word、TXT、Markdown等格式的文档。 直接点击上传或拖拽文件。这是最常用、最直接的方式。
添加在线URL 资料来源于网页，比如产品官网、帮助文档、博客文章等。 输入网页链接，平台会自动抓取内容。可以添加单个或多个链接。

3. 设置处理规则并完成创建
添加文档后，点击下一步，你会看到一个**“数据处理”**的界面。这里通常使用默认的“自动分段与清洗”选项即可。平台会自动完成我们之前聊过的所有技术步骤：

  • 文档解析：提取文件或网页中的文字。
  • 文本分割（切片）：按语义将长文本拆分成合适的段落。
  • 向量化与索引：将文本转换成向量并存储，建立可快速检索的索引。

处理完成后，你的知识库就建好了。

💡 核心：关联知识库并开始使用

知识库本身不会主动回答问题，你需要让它和一个 “Bot”（智能体） 结合：

  1. 创建一个新的Bot，或编辑一个已有的Bot。
  2. 在Bot的编辑页面，找到「技能」或「知识库」模块，点击“添加知识库”，选择你刚刚创建的那个。
  3. 保存并发布这个Bot。现在，当你向这个Bot提问时，它就会自动从你的知识库中查找信息来回答你。

4.3 Function Calling

4.3.1 🔧 Function Calling 工作原理

你可以把它理解成大模型（AI大脑）的“手脚”。AI大脑自己不能执行实际动作（比如查天气、发邮件），但它可以识别你的意图，并指挥对应的“手脚”（函数）去完成。

flowchart LR
    A[用户提问<br>“北京天气怎么样？”] --> B(大模型理解)
    
    B --> C{决策}
    C -- 需要真实数据 --> D[“调用“查天气”函数<br>（附上参数：北京）”]
    C -- 直接可回答 --> E[直接生成回复]
    
    D --> F[外部系统/API<br>执行函数]
    F --> G[获取真实结果<br>“北京：25°C，晴”]
    
    G --> B
    E --> H[最终回复]
    B -.-> H

一句话总结：Function Calling 让大模型从“只会说”变成了“能指挥别人去做”。

4.3.2 🖥️ 在 Coze 网页上如何使用 Function Calling？

在Coze上，这个过程被大大简化，你几乎不需要写代码。主要通过给 Bot（智能体）添加“插件”或“工作流” 来实现。

1. 核心概念：插件就是现成的“函数”

Coze 平台集成了大量现成的插件（如：查天气、搜网页、发邮件、订机票等）。你只需要：

  • 在编辑 Bot 时，点击「插件」。
  • 在插件商店里搜索添加（例如“天气”）。
  • 添加后，当用户问相关问题时，Bot 就会自动调用这个插件函数。
  • 你也可以在提示词通过 {插件的名字} 符号来调用插件
2. 高级操作：用“工作流”自定义复杂函数

如果你需要更复杂或个性化的功能，可以使用「工作流」。

  • 工作流像一个可视化的编程界面，你可以通过拖拽组件来连接：输入 -> 调用一个API或内部逻辑 -> 输出。
  • 把这个工作流发布成一个“节点”，它就会出现在你的 Bot 技能里，成为一个可以被调用的自定义函数。
3. 实际效果

配置好后，当你的用户问：“帮我查一下上海今天下午会不会下雨？”

  • 你的 Bot 会自动识别意图，调用你配置的“天气查询”插件或工作流。
  • 插件会执行查询，返回数据。
  • Bot 会组织语言回复：“上海今天下午多云，降雨概率10%，气温22-26°C。”

1.第一步：准备天气API

你需要一个提供天气数据的服务。以高德开放平台为例：

  • 访问其官网，注册开发者账号。
  • 在控制台创建一个新应用，获取专属的 API Key（一串密钥），这是调用天气接口的凭证。
  • 找到天气查询API的文档，记下它的请求地址（URL）和需要的参数（如城市、返回格式等）。
2.第二步：在Coze创建新插件
  • 在Coze主页进入“工作空间” > “资源库” > “插件”。
  • 点击“新建插件”，类型选择 “云侧插件 - 基于已有服务创建”。这表示我们要接入一个已存在的API服务。
3.第三步：填写API服务信息
  • 在弹出的表单中，填写插件名称（如“我的天气查询”）和描述。
  • 在“服务配置”里，粘贴你从高德API文档中找到的基础URL。
  • 在“Headers”（请求头）中，通常需要添加一个 key 字段，值就是你申请到的 API Key。
  • 授权方式一般选择“不需要授权”。
4.第四步：创建与配置“工具”
  • 保存上一步后，进入插件详情页，点击“创建工具”。一个插件里可以包含多个“工具”，这里我们创建一个叫“查询天气”的工具。
  • 工具路径：填写天气API的具体路径，比如 /v3/weather/weatherInfo。这会被拼接到基础URL后面形成完整地址。
  • 请求方法：选择 GET（高德天气API通常用此方法）。
  • ⭐配置参数：根据API文档，添加必需的查询参数，分输入参数和输出参数要填。例如，一个 city 参数（城市名），并设定它为“必填”，这样用户必须提供城市名才能查询。
5.第五步：测试运行
  • 所有参数填好后，页面通常有“试运行”或“调试”按钮。
  • 输入一个测试值（如“北京”），点击运行。如果配置正确，右侧会返回该城市的天气JSON数据。
6.第六步：发布并使用
  • 测试成功后，启用并“发布”这个插件。
  • 之后，在你创建或编辑一个Bot（智能体） 时，就可以在“技能”或“插件”区域找到并添加你自制的这个天气插件了。

4.4 工作流

4.4.1 工作流的原理

工作流就像是一条生产流水线。智能体在处理用户的问题时，会按照预设的步骤依次执行。例如，当用户输入美食推荐需求后，智能体先通过意图识别确定用户的需求，然后从知识库中查找相关信息，再调用搜索工具获取最新数据，最后根据这些信息生成推荐结果。在扣子平台，你可以通过可视化的方式配置工作流，设置每个步骤的执行顺序和条件。

在Coze中，工作流是一个强大的工具，它让你能够像搭积木一样，通过可视化的方式，将AI大语言模型、插件、代码等功能模块组合成自动化的业务流程。这非常适合处理那些步骤较多、对结果准确性或格式有严格要求的复杂任务。

4.4.2 🔧 工作流节点分类

  • 工作原理：你可以把工作流理解为一个自动化流水线。每个节点就是流水线上的一个“工位”，负责一项特定工作（如理解问题、调用工具、判断逻辑）。数据像“工件”一样从一个节点流向下一个节点，直至最终输出结果。

  • 节点分类：工作流由多种节点构成，主要分为以下几类：

    节点类别 主要节点举例 核心功能
    输入/输出 开始节点、结束节点 定义流程的输入参数和最终输出结果。⭐开始节点：点击节点，选择开始，点击user_input。⭐结束节点：写在里面作为承接，并且选即可你对应接收的信息
    逻辑控制 条件/选择器节点、循环节点 实现条件分支（if/else）和循环/批量处理。
    功能处理 大语言模型(LLM)节点、插件节点、代码节点、知识库节点 这是能力的核心：LLM用于理解与生成，插件用于调用外部服务，代码节点执行定制逻辑，知识库节点检索专属信息。
    实用工具 文本处理节点、变量聚合节点、HTTP节点 进行文本拼接拆分、合并多个分支的结果、直接调用外部API等。
4.4.3 📝 工作流的四种主要形式

基于上述节点，可以构建出几种典型的工作流形式：

  1. 多步骤任务：将复杂任务线性拆解。例如，“分析报告生成”工作流：输入主题 → LLM生成大纲 → 联网搜索资料 → 整理成文。
  2. 条件分支：使用条件/选择器节点，根据不同输入走不同路径。例如，在客服场景中，根据用户意图识别结果，分别跳转到“查询订单”或“退货流程”分支。
  3. 批量处理：利用循环节点或批处理节点，自动对一批数据（如一个列表）执行相同操作。例如，批量处理多张图片、分析多个新闻链接。
  4. 数据转换：通过代码节点或文本处理节点，清洗、计算或转换数据格式。例如，将从API获取的原始JSON数据，提取关键字段并重新组装。
4.4.4 🛠️ Coze创建工作流标准流程

无论创建哪种形式的工作流，都遵循一个通用的五步流程：

  1. 创建与初始化：在个人或团队空间中点击“+资源” → 选择“工作流”。输入名称（需用英文/数字）和描述。
  2. 拖拽编排节点：从左侧节点库中，将需要的节点拖到中间画布上。通常从设置“开始节点”的输入参数（如用户问题、图片等）入手。
  3. 连接并配置节点：用连线连接节点，表示数据流向。点击每个节点进行详细配置，这是关键步骤，例如：
    • LLM节点：编写精确的提示词（Prompt），并引用上游变量。
    • 插件节点：选择具体插件并配置其输入。
    • 条件节点：设置判断规则。
  4. 调试与试运行：点击画布下方的“试运行”按钮。在右侧调试面板输入测试数据，逐步运行检查每个节点的输入/输出是否符合预期，这是排查问题的主要方式。
  5. 发布与集成：调试成功后，点击“发布”。发布后的工作流可以像插件一样，被添加到智能体（Bot） 的技能中，当用户对话触发时即可调用。

5. 意图识别插件的构建

5.1 明确意图分类

对于美食推荐助手，常见的意图可能有“查询餐厅推荐”“查询美食信息”“查询餐厅评分”等。

5.2 规则制定

根据不同的意图，制定相应的规则。例如，当用户输入包含“推荐餐厅”“哪里吃饭”等关键词时，判断为“查询餐厅推荐”意图。

    A[“用户输入<br>一句话”] --> B[“意图识别模型<br>（大脑）”]
    B --> C{“判断意图属于哪一类”}
    
    C -- “查询类” --> D[“例: 今天天气如何？<br>→ 匹配‘天气查询’意图”]
    C -- “操作类” --> E[“例: 播放周杰伦的歌<br>→ 匹配‘音乐播放’意图”]
    C -- “闲聊类” --> F[“例: 你好吗？<br>→ 匹配‘日常问候’意图”]
    
    D --> G[“触发对应插件/工作流<br>（执行具体任务）”]
    E --> G
    F --> G

5.3 在扣子平台实现

在扣子平台，可以通过编写简单的规则代码或者使用平台提供的可视化配置工具来实现意图识别。配置完成后，智能体在接收到用户输入时，会先进行意图识别，然后根据不同的意图执行相应的操作。

6. 测试与迭代

6.1 快速测试清单

  • 输入“我想吃川菜，预算200元，在上海”，检查是否能正确返回符合条件的餐厅推荐信息。
  • 输入“某餐厅的评分”，检查是否能正确调用工具返回评分。

6.2 优化逻辑

如果回答不准确或不完整，可通过以下方式优化：

  • 增加示例：在Prompt中添加更多的示例，让智能体更好地理解任务要求。
  • 调整任务边界：明确任务的范围，避免智能体回答过于宽泛或偏离主题。
  • 优化工具调用参数：调整搜索关键词、知识库的筛选条件等，提高信息获取的准确性。
