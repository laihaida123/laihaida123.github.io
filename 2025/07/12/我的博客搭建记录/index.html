✅ 什么是 Algolia 搜索？

Algolia 是一个提供快速、强大全文搜索服务的平台。Hexo + Algolia 可以实现比本地搜索（local search）更快更智能的搜索体验，尤其适合文章较多的博客。

✅ 二、配置 fields 字段（很重要）

Algolia 配置完整步骤详解
1. 注册 Algolia 账号
  • 访问官网：https://www.algolia.com/
  • 点击右上角“Sign up”注册账号
2. 创建一个新的索引（Index）
  • 登录后，在控制台首页点击 “New Application”（新建应用）
  • 输入一个应用名称，比如：my-blog-search
  • 创建完成后进入这个应用管理页面
  • 点击左侧菜单的 “Indices”，点击 “Add new index”
  • 输入索引名称，比如：hexo_blog_index，点击确认
3. 获取 API Key 和 App ID
  • 在 Algolia 控制台，点击左侧的 “API Keys” 菜单
  • 你会看到：
名称 说明
Application ID App ID，Hexo 配置中用到
Write API Key 管理权限密钥，用于上传数据
Search API Key 用于前端搜索，权限有限

折腾了一整个下午，终于把 Hexo 和 Algolia 搜索搞定了。看似简单的几行配置，实则每一个字段名都可能藏着坑。最开始报错看不懂、路径出错、插件搞混、API Key 放错……真的是一边查文档一边试错，有时候一个缩进都能搞得人抓狂。

但也正是这种“踩坑→解决→继续”的过程，才是折腾的意义吧。看着搜索框终于能响应关键词、Algolia 返回的数据瞬间加载出来的那一刻，有种“技术点亮生活”的小小成就感。

这让我想起刚开始接触前端的时候，能让页面动起来就觉得很酷。现在一步步集成第三方搜索服务、改配置、部署上线，每一步都走得更稳了。

虽然花了不少时间，但换来的是更多信心：以后遇到问题，不怕了，查查文档、调调代码，总能解决。

每每出现问题，我都想起sing，我们好久没见面了，下午的时候一直在想她，（（她应该不会看到我这段话吧?__?））

hexo博客添加评论系统

安装

进入Giscus的安装界面点击install安装

官网配置

选择一个GitHub仓库用于储存评论数据。

  1. 该仓库是公开的；
  2. giscus app 已安装；
  3. Discussions 功能已在你的仓库中启用。 进入对应的仓库，点击右上角的 Settings，往下翻到Features，勾选Discussions。

个性化设置

【页面 <-> Discussion映射关系】

建议选择title

【Discussion分类】

建议选择Announcements，并勾选下面的“只搜索该分类中的discussion”

【特性】

建议勾选“启用主帖子上的反应（可以给博客文章贴表情）”，“将评论框放在评论上方”和“懒加载评论（评论的加载将延迟到用户滚动到评论容器附近，有助于博客界面的快速加载，是一种前端显示的优化手段）”

【主题】

建议勾选“用户偏好的色彩方案”
