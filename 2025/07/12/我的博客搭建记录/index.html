✅ 什么是 Algolia 搜索？

Algolia 是一个提供快速、强大全文搜索服务的平台。Hexo + Algolia 可以实现比本地搜索（local search）更快更智能的搜索体验，尤其适合文章较多的博客。

✅ 二、配置 fields 字段（很重要）

Algolia 配置完整步骤详解
1. 注册 Algolia 账号
  • 访问官网：https://www.algolia.com/
  • 点击右上角“Sign up”注册账号
2. 创建一个新的索引（Index）
  • 登录后，在控制台首页点击 “New Application”（新建应用）
  • 输入一个应用名称，比如：my-blog-search
  • 创建完成后进入这个应用管理页面
  • 点击左侧菜单的 “Indices”，点击 “Add new index”
  • 输入索引名称，比如：hexo_blog_index，点击确认
3. 获取 API Key 和 App ID
  • 在 Algolia 控制台，点击左侧的 “API Keys” 菜单
  • 你会看到：
名称 说明
Application ID App ID，Hexo 配置中用到
Write API Key 管理权限密钥，用于上传数据
Search API Key 用于前端搜索，权限有限

序

在加入这个之前，需要了解一些东西：

什么是Pjax？

Pjax是提升网页加载速度和用户体验的前端技术，它通过结合 Ajax 和 HTML5 History API（pushState）实现页面局部刷新，同时保持浏览器地址栏的 URL 更新和前进/后退功能。这样可以不用重复加载相同的资源（css/js）， 从而提升网页的加载速度。

大白话：

Pjax 就是，点链接时，只刷新网页的「一小块内容」，其他部分不变（比如顶部导航栏、侧边栏），同时网址会变，还能用浏览器的「前进」「后退」按钮。

🌰：

用 Ajax 偷偷换内容 + 用浏览器历史记录改网址 = 假装跳转了页面，其实只换了个零件。

（现在用的知乎/小红书等App，点开新页面时其实就用类似技术）

加入：

当用户点击链接，通过 ajax 更新页面需要变化的部分，然后使用 HTML5 的 pushState 修改浏览器的 URL 地址。

这样可以不用重复加载相同的资源（css/js）， 从而提升网页的加载速度。

PWA

序：

先要了解下PWA是什么

官方：

PWA （Progressive Web App，渐进式 Web 应用）是一种结合了网页和原生应用优点的技术，让网页可以像手机 App 一样使用。它的核心理念是：用 Web 技术（HTML、CSS、JavaScript）构建应用，但拥有接近原生 App 的体验。

核心技术：

Service Worker（后台运行的脚本，支持离线缓存、推送等）

Web App Manifest（定义图标、启动画面、主题色等，让浏览器知道这是一个可安装应用）

HTTPS（PWA 必须在安全环境下运行）

大白话：

PWA 就是让网站像手机 App 一样用的技术。

它的好处是：

  • 可以安装到手机桌面
  • 没网也能用（离线）

简单理解：PWA = 网站 + App 的结合。

Inject 是 Butterfly 主题（v2.3.0+）提供的一个“插入点”功能。

  • 它允许你在不直接修改主题源码的情况下，把自己的 JS、CSS、meta 等标签插入到页面 HTML 的特定位置
  • 位置有两个：
    1. head（</head> 之前）
      适合放样式、meta 标签、SEO 代码、外部字体、第三方库等
    2. bottom（</body> 之前）
      适合放脚本、统计代码、广告代码等

请注意：以标准的 html 格式添加内容
